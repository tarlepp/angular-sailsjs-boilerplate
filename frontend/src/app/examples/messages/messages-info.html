General info

Example page to demonstrate how this boilerplate is handling errors with $http and $sailsSocket requests. Note that this magic is done automatic via error interceptor, so you don't have to do anything extra for error handling. Actual error handling is done via error interceptor that is attached to $http and $sailsSocket services.

This interceptor will catch all errors from $http and $sailsSocket request and show those to user via message service. Note that shown message depends on actual error response from backend.

Functions in this example

Custom messages
Simple example to trigger different types of messages; info, success, warning and error with specified title and actual message.
$http / $sailsSocket
Examples to demonstrate invalid URL and not found record. These are handled automatic by ErrorInterceptor.
